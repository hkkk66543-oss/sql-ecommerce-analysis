# 指标口径说明

## 1. 经营总览指标

### GMV

GMV 指平台订单商品销售金额总和。

计算口径：

```sql
GMV = SUM(price)
```

说明：本项目中的 GMV 主要统计商品成交金额，不包含运费。

---

### 订单数

订单数指平台有效订单数量。

计算口径：

```sql
订单数 = COUNT(DISTINCT order_id)
```

说明：以订单 ID 去重统计。

---

### 客单价 AOV

AOV 指平均每笔订单贡献的商品销售金额。

计算口径：

```sql
AOV = GMV / 订单数
```

---

### 总运费

总运费指所有订单明细中的运费金额汇总。

计算口径：

```sql
总运费 = SUM(freight_value)
```

---

### 运费占比

运费占比用于衡量运费成本相对于商品销售金额的比例。

计算口径：

```sql
运费占比 = 总运费 / GMV
```

---

### 月度 GMV

按订单下单月份统计的 GMV。

计算口径：

```sql
月度GMV = 按 order_purchase_timestamp 所在月份汇总 SUM(price)
```

---

### 月度订单数

按订单下单月份统计的订单数量。

计算口径：

```sql
月度订单数 = 按 order_purchase_timestamp 所在月份统计 COUNT(DISTINCT order_id)
```

---

### 支付方式订单数

按支付方式统计订单支付记录数量。

计算口径：

```sql
支付方式订单数 = COUNT(*)
```

说明：用于分析 credit_card、boleto、voucher、debit_card 等支付方式分布。

---

### 订单状态订单数

按订单状态统计订单数量。

计算口径：

```sql
订单状态订单数 = COUNT(DISTINCT order_id)
```

说明：用于分析 delivered、shipped、canceled、unavailable、invoiced、processing 等状态分布。

---

## 2. 物流履约指标

### 平均配送天数

指从用户下单到订单实际签收之间的平均天数。

计算口径：

```sql
平均配送天数 = AVG(实际签收时间 - 下单时间)
```

---

### 商家发货天数

指从订单确认后到商家发货之间的平均天数。

计算口径：

```sql
商家发货天数 = AVG(实际发货时间 - 订单确认时间)
```

---

### 承运配送天数

指从商家发货到用户实际签收之间的平均天数。

计算口径：

```sql
承运配送天数 = AVG(实际签收时间 - 实际发货时间)
```

---

### 延迟送达率

指实际签收时间晚于预计送达时间的订单占比。

计算口径：

```sql
延迟送达率 = 延迟送达订单数 / 总订单数
```

判断逻辑：

```sql
实际签收时间 > 预计送达时间
```

---

### 平均延迟天数

指延迟送达订单中，实际签收时间超过预计送达时间的平均天数。

计算口径：

```sql
平均延迟天数 = AVG(实际签收时间 - 预计送达时间)
```

仅统计延迟送达订单。

---

### 发货延迟率

指商家发货时间晚于平台设定或业务判断标准的订单占比。

计算口径：

```sql
发货延迟率 = 发货延迟订单数 / 总订单数
```

说明：该指标主要用于衡量商家侧履约效率。

---

### 平均评分

指订单评价分数的平均值。

计算口径：

```sql
平均评分 = AVG(review_score)
```

说明：原始评价分数为 1-5 分整数，聚合后平均分可能出现小数。

---

### 差评率

差评率指评分为 1-2 分的订单占比。

计算口径：

```sql
差评率 = 差评订单数 / 有评价订单数
```

判断逻辑：

```sql
review_score <= 2
```

---

### 中评率

中评率指评分为 3 分的订单占比。

计算口径：

```sql
中评率 = 中评订单数 / 有评价订单数
```

判断逻辑：

```sql
review_score = 3
```

---

### 好评率

好评率指评分为 4-5 分的订单占比。

计算口径：

```sql
好评率 = 好评订单数 / 有评价订单数
```

判断逻辑：

```sql
review_score >= 4
```

---

### 配送时长分组

根据订单从下单到签收的总配送天数进行分组。

分组口径：

```text
0-3 days
4-7 days
8-15 days
16-30 days
30+ days
```

用途：分析配送时长变化对平均评分、差评率和好评率的影响。

---

### 地区延迟送达率

按客户所在州统计延迟送达订单占比。

计算口径：

```sql
地区延迟送达率 = 该地区延迟送达订单数 / 该地区总订单数
```

---

### 商家送达延迟率

按商家维度统计延迟送达订单占比。

计算口径：

```sql
商家送达延迟率 = 该商家延迟送达订单数 / 该商家总订单数
```

说明：用于识别高物流风险商家。

---

## 3. 用户运营指标

### 用户数

用户数指平台去重后的用户数量。

计算口径：

```sql
用户数 = COUNT(DISTINCT customer_unique_id)
```

---

### 复购用户数

复购用户数指购买次数大于等于 2 次的用户数量。

计算口径：

```sql
复购用户数 = COUNT(DISTINCT customer_unique_id WHERE 购买次数 >= 2)
```

---

### 复购率

复购率指复购用户数占总用户数的比例。

计算口径：

```sql
复购率 = 复购用户数 / 总用户数
```

---

### 用户购买频次

按用户购买订单次数进行分组。

分组口径：

```text
1 order
2 orders
3 orders
4+ orders
```

用途：分析平台用户是否以一次性购买为主，以及复购用户规模。

---

### 用户 GMV

指单个用户贡献的商品销售金额总和。

计算口径：

```sql
用户GMV = SUM(price)
```

---

### 用户客单价

指单个用户平均每笔订单的商品销售金额。

计算口径：

```sql
用户客单价 = 用户GMV / 用户订单数
```

---

### RFM 模型

RFM 是用户价值分层模型，由三个维度构成：

```text
R：Recency，最近一次购买距分析基准日的天数
F：Frequency，用户购买次数
M：Monetary，用户累计消费金额
```

---

### Recency

Recency 指用户最近一次购买距离分析基准日的天数。

计算口径：

```sql
Recency = 分析基准日 - 用户最近一次购买日期
```

说明：数值越小，表示用户越近期活跃。

---

### Frequency

Frequency 指用户累计购买订单次数。

计算口径：

```sql
Frequency = COUNT(DISTINCT order_id)
```

---

### Monetary

Monetary 指用户累计消费金额。

计算口径：

```sql
Monetary = SUM(price)
```

---

### RFM 用户分层

基于 Recency、Frequency、Monetary 三个维度对用户进行分层。

本项目用户分层包括：

```text
高价值用户
重要保持用户
重要发展用户
重要挽留用户
潜力用户
新用户
一般用户
流失风险用户
```

---

### RFM 用户数占比

指某一用户分层的用户数量占总用户数量的比例。

计算口径：

```sql
用户数占比 = 该分层用户数 / 总用户数
```

---

### RFM GMV 贡献率

指某一用户分层贡献的 GMV 占平台总 GMV 的比例。

计算口径：

```sql
GMV贡献率 = 该分层GMV / 总GMV
```

---

### 高价值用户数

高价值用户数指 RFM 分层中被识别为高价值用户的用户数量。

计算口径：

```sql
高价值用户数 = COUNT(DISTINCT customer_unique_id WHERE 用户分层 = '高价值用户')
```

---

### 高价值用户 Top 品类 GMV

指高价值用户在不同商品品类上的消费金额。

计算口径：

```sql
高价值用户品类GMV = 按商品品类汇总高价值用户 SUM(price)
```

用途：识别高价值用户偏好的核心商品品类。

---

### 流失风险用户数

流失风险用户数指 RFM 分层中被识别为流失风险用户的用户数量。

计算口径：

```sql
流失风险用户数 = COUNT(DISTINCT customer_unique_id WHERE 用户分层 = '流失风险用户')
```

---

### 流失风险用户召回优先级

根据流失风险用户的消费金额、购买频次、最近购买间隔等指标，将用户划分为不同召回优先级。

本项目召回优先级包括：

```text
低优先级召回
中优先级召回
```

用途：辅助运营人员区分不同流失风险用户的召回价值。

---

## 4. 评价口径说明

### 原始评分

原始评价分数来自订单评价表 `review_score` 字段，取值范围为 1-5 分整数。

---

### 平均评分为什么会出现小数

由于看板中的评分多为聚合后的平均评分，因此会出现 4.08、4.33、2.55 等小数。

计算口径：

```sql
平均评分 = AVG(review_score)
```

---

### 好评率 + 中评率 + 差评率

理论上：

```text
好评率 + 中评率 + 差评率 ≈ 100%
```

若存在 0.01 左右误差，通常来自百分比四舍五入。

---

## 5. 数据层级说明

### ODS 层

ODS 层为原始数据层，主要存储从 Kaggle 原始 CSV 导入 MySQL 后的基础业务表。

---

### DWD 层

DWD 层为清洗明细层，用于整合订单、商品、支付、用户、商家、评价、物流等多表数据，形成可分析的明细宽表。

---

### ADS 层

ADS 层为指标汇总层，用于生成 Power BI 看板直接使用的经营、物流、用户运营指标表。
